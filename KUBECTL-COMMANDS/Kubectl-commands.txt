Hereâ€™s a comprehensive list of common kubectl commands and their usages for managing Kubernetes clusters. These commands cover a range of functionalities including managing pods, deployments, services, and more.

1. Cluster Management
Check Cluster Info:


kubectl cluster-info
View Nodes:


kubectl get nodes
View Node Details:


kubectl describe node <node-name>
Check Contexts and Switch Contexts:


kubectl config get-contexts
kubectl config use-context <context-name>
Get Cluster Config:


kubectl config view
2. Pod Management
List Pods:


kubectl get pods
View Pod Details:


kubectl describe pod <pod-name>
Get Pod Logs:


kubectl logs <pod-name>
Execute Command in Pod:


kubectl exec -it <pod-name> -- <command>
Delete Pod:


kubectl delete pod <pod-name>
3. Deployment Management
List Deployments:


kubectl get deployments
View Deployment Details:


kubectl describe deployment <deployment-name>
Scale Deployment:


kubectl scale deployment <deployment-name> --replicas=<number>
Update Deployment:


kubectl set image deployment/<deployment-name> <container-name>=<new-image>
Rollout Status:


kubectl rollout status deployment/<deployment-name>
Rollout History:


kubectl rollout history deployment/<deployment-name>
Rollout Undo:


kubectl rollout undo deployment/<deployment-name>
4. Service Management
List Services:


kubectl get services
View Service Details:


kubectl describe service <service-name>
Delete Service:


kubectl delete service <service-name>
5. Namespace Management
List Namespaces:


kubectl get namespaces
View Namespace Details:


kubectl describe namespace <namespace-name>
Delete Namespace:


kubectl delete namespace <namespace-name>
6. Configuration Management
List ConfigMaps:


kubectl get configmaps
View ConfigMap Details:


kubectl describe configmap <configmap-name>
Delete ConfigMap:


kubectl delete configmap <configmap-name>
List Secrets:


kubectl get secrets
View Secret Details:


kubectl describe secret <secret-name>
Delete Secret:


kubectl delete secret <secret-name>
7. Persistent Volume Management
List Persistent Volumes:


kubectl get pv
View Persistent Volume Details:


kubectl describe pv <pv-name>
Delete Persistent Volume:


kubectl delete pv <pv-name>
8. Job and CronJob Management
List Jobs:


kubectl get jobs
View Job Details:


kubectl describe job <job-name>
Delete Job:


kubectl delete job <job-name>
List CronJobs:


kubectl get cronjobs
View CronJob Details:


kubectl describe cronjob <cronjob-name>
Delete CronJob:


kubectl delete cronjob <cronjob-name>
9. Resource Management
Get Resources by Type (e.g., Pods, Services, Deployments):


kubectl get <resource-type>
Describe Resource by Type (e.g., Pod, Service):


kubectl describe <resource-type> <resource-name>
Delete Resource by Type (e.g., Pod, Service):


kubectl delete <resource-type> <resource-name>
10. Cluster Health and Troubleshooting
Check Cluster Nodes:


kubectl top nodes
Check Pod Resource Usage:


kubectl top pods
View Events:


kubectl get events
These commands help manage and interact with a Kubernetes cluster, enabling you to perform various tasks from deployment and scaling to troubleshooting and monitoring.
